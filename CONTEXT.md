# Beat Typer

把 Beat Saber 譜面轉成 3D 節奏打字練習:每顆音符依「顏色/欄/列」唯一對應到一個鍵盤字母,玩家在「長得像鍵盤」的高速公路上敲字。本檔是詞彙表,只定義概念,不含實作細節。

## Beat Saber 譜面座標(輸入端詞彙)

**欄 (Column)**:
音符的水平位置,`0~3`,玩家視角由左到右(col0 最左、col3 最右)。v2 `_lineIndex` / v3 `x`。
_Avoid_: lineIndex(那是原始欄位名,對外一律講「欄」)

**列 (Row)**:
音符的垂直位置,`0~2`,由下到上(row0 下、row1 中、row2 上)。v2 `_lineLayer` / v3 `y`。
_Avoid_: layer, lineLayer

**顏色 (Color)**:
音符的紅/藍屬性,決定由哪隻手負責:紅=左手、藍=右手。v2 `_type` / v3 `c`。
_Avoid_: type, cutDirection(切法方向本作不使用)

**變速 (BPM change)**:
曲中 BPM 的分段變化,決定 beat→秒的換算。以「BPM 時間線」表示——排序的 `[{beat, bpm}]`,區段 `[bᵢ, bᵢ₊₁)` 以該段 BPM 播放。來源:v3 `bpmEvents` / v2 `_customData._BPMChanges`;二者凌駕 Info.dat 的 BPM(後者僅為顯示值與基準)。見 docs/adr/0009。
_Avoid_: tempo, bpmEvents(對外一律講「變速 / BPM 時間線」)

## 映射(核心概念)

**映射 (Mapping)**:
把一顆音符決定成一個鍵盤鍵的規則。**手由顏色決定**(唯一保留的位置語義);手指與排(即最終的鍵)不由欄/列決定,而交給「鍵指派」為打字練習平衡而選。

**鍵指派 (Key Assignment)**:
在單場內把每顆音符指派到一鍵、以打字練習價值(而非還原 Beat Saber 空間)為目標的過程。可決定性:同一張譜+難度永得同一組指派(可 replay);不同譜各自平衡,故跨譜不保證同一音符對到同一鍵。

**教學權重 (Teaching Weight)**:
每個鍵在打字練習中的期望佔比:家排 > 上排 > 下排、食指/中指 > 無名/小指。鍵指派朝此權重攤平各鍵出現量;權重皆為正,故足夠長的譜自然趨向全鍵覆蓋。

**可玩性 (Playability)**:
鍵指派的硬底線:同一手、時間相鄰過近的音符不得指派到同一手指。平衡只在滿足此底線的選項中進行。

**鍵群 (Key Group)**:
鍵指派可用鍵池的一個預設子集,供針對性練習(如只練家排、只練食指/中指)。一律**雙手對稱**的排/指子集——兩手都保有鍵,故不改變顏色→手、不濾音符、不動判定分母,只是在更小的池上照常攤平/守可玩性。`all`(全鍵)為預設、等同無限制。為跨場偏好,經設定層持久化。見 docs/adr/0011。
_Avoid_: 練習模式, 字表(那是詞庫不是鍵群)

**手 (Hand)**:
左或右,由顏色決定。左右手各自的總負荷等於該譜紅/藍音符的比例,不由鍵指派調整(手均衡為顯式非目標)。

**手指 (Finger)**:
一隻手的小指/無名/中指/食指之一,是**被指派鍵**的屬性(由鍵指派決定,不再由欄推導)。

**排 (Bank)**:
鍵盤的上排/家排/下排,是**被指派鍵**的屬性(由鍵指派決定,不再由列推導)。

**內側鍵 (Inner Key)**:
兩食指內側的鍵(左 T/G/B、右 Y/H/N)。與其他鍵無異,是鍵指派可用鍵池的一部分(各帶自己的教學權重)。

**疊放 (Stack)**:
同一手、彼此相距 **< 1/8 beat** 的多顆音符;在 Beat Saber 意義上是「一次揮砍的強調」。以錨點制成群,整組收斂成一顆音符(收斂為節奏/可玩性考量);收斂後的那顆與其他音符一樣交給鍵指派。

**連打 (Burst)**:
同一手、相距 **≥ 1/8 beat** 的連續音符;是原譜本來就有的節奏,各自保留為獨立音符(與「疊放」互斥),各自交給鍵指派。

## 載入(輸入端接縫)

**ChartSource**:
一個譜面來源(內建 / zip / CustomLevels 資料夾),能 `listSongs()` 列出其中的歌。三種來源共用此介面,compileChart 對來源無感。

**SongHandle**:
單一首歌,提供惰性的 `readFile(name)`(單檔讀取、檔名大小寫寬容)。選定歌/難度後才讀對應檔與音訊。

**parseInfo**:
把 Info.dat 文字淺解析成 `{ bpm, songTimeOffset, audioFilename, difficulties }` 的純函式。供難度選單與 compileChart 共用,使讀檔(I/O)留在編排層。

**RawMapFiles**:
交給 compileChart 的單首歌原始檔案(檔名→未解析文字);compileChart 自己 parse 與正規化,是唯一正規化點。

## 攤平後的輸出

**TypingChart**:
`compileChart` 的輸出;一條依時間排序的按鍵時間軸。同一手同一時刻至多一鍵(同手疊放收斂為一鍵、同手連打各自保留);跨手可同時(紅+藍同拍共用時間)。是渲染與判定共同消費的唯一資料來源。
_Avoid_: keymap, sequence

**音符 (Note)**:
TypingChart 的一個元素:`{ tSec, key, kind }`(外加渲染中繼:hand/finger/bank)。`key` 存**實體按鍵碼**(`KeyboardEvent.code`,如 `"KeyF"`、`"Semicolon"`),不是字元;`kind` 為 `press` 或 `hold`(hold 帶 `holdEndSec`)。
_Avoid_: event, keystroke

**字形 (Glyph)**:
一個 `key`(按鍵碼)給玩家看的顯示字元(`"Semicolon"`→`;`)。永遠由 `glyphOf(code)` 即時推導,不存進 Note。

**compileChart**:
把原始譜面檔 + 難度 + config 轉成 TypingChart 的純函式。可決定性,不含 I/O、音訊或渲染。主測試接縫。

## 遊玩(輸出端詞彙)

**高速公路 (Highway)**:
3D 遊玩畫面;音符朝判定平面飛來。左手音符在左、右手在右。

**道 (Lane)**:
高速公路上音符飛來的水平位置,共 **10 道**照實體鍵盤由左到右排:左手小指/無名/中指/食指、中央左內側(TGB)/右內側(YHN)、右手食指/中指/無名/小指。道與高度(Y)皆依**該音符被指派到的鍵**在鍵盤上的位置決定(上排飛高、下排飛低),故畫面位置 = 該鍵在鍵盤上的(左右, 上下)。內側鍵佔獨立道。

**判定平面 (Judgment Plane)**:
音符抵達、玩家該敲鍵的那一刻的平面;在近端呈現 **10 欄 × 3 列的靜態目標格線**(一張鍵盤輪廓),讓玩家預見落點與手指預備位。

**judge**:
把 TypingChart + 帶時戳按鍵事件 + config 轉成判定與 summary(準確率/combo/評級)的純函式。次測試接縫。即時路徑與批次共用同一組增量原語(`Judger`:`press` / `expiry`),邏輯單一真相來源。

**判定 (Judgment)**:
單顆音符的結果:Perfect(窗內近)/ Good(窗內遠)/ Miss(出窗、未敲到、窗內敲錯鍵,或長按提早放開)。長按與一般音符同為**一個判定單位**(同權計分、combo +1)。

**長按 (Hold)**:
一種音符,玩家須在頭部按對鍵、持續按住整個持續段、撐到尾部附近再放開才算命中。手由顏色決定、鍵由鍵指派決定(與一般音符無異);來源為 v3 弧線,v2 弧線不支援。無對應長按的放開一律忽略(不罰)。
_Avoid_: slider, arc, 弧線(對外一律講「長按」)

**持續段 (Sustain)**:
長按從頭部(`tSec`)到尾部(`holdEndSec`)須持續按住的區段;在高速公路上呈現為沿走廊的長條。

**破壞點 (Break Point)**:
長按尾部容差的唯一界線 = `holdEndSec − goodSec`。在此之前放開 = 沒撐住(判 Miss、斷 combo);撐過此點(不論何時放或按到底)即鎖定命中。

**鎖定 (Secure)**:
長按撐過破壞點後定案為命中,維持頭部判得的 Perfect/Good。從不放開者於尾部時間自動鎖定,不懸置。

**多餘按鍵 (Extra press)**:
附近無音符在窗內時的按鍵。**完全不罰**:不斷 combo、不計入準確率,僅計數顯示。

**評級 (Grade)**:
由準確率換算的總評(S/A/B/C/D)。準確率 = Σ權重(Perfect 1 / Good 0.5 / Miss 0)/ 音符數。
