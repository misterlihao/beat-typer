# 22 — 自製譜面系統

> 來源:使用者需求(2026-07-12,backlog 記錄;2026-07-13 grill 收斂第一階段)。
> **狀態:階段一規格已收斂(可動手);階段二仍待評估(等階段一驗證可玩性)。**

## 問題 / 動機

目前所有譜面都來自 Beat Saber(內建 / zip / BSR),經 `compileChart` 轉成 TypingChart。缺一條「自己做譜」的路:讓使用者不依賴既有 Beat Saber 圖的**音符排列**,也能自己排要練的節奏。這開啟「Beat Typer 作為獨立打字練習工具」的可能,而不只是 Beat Saber 譜面的轉換器。

## 為何分階段

「完整自製譜面系統」(自帶音訊 / 時鐘 / 成績身分 / 編輯器 / 分享)一次做會大幅擴張架構邊界,而**這套可玩性到底能提升多少、值不值得加入系統仍未知**。因此先做一個成本低、能實測可玩性的**階段一**;確認有價值再投入階段二。

階段一的核心待驗證假設:**「自己排音符 + 用『強調』這個新表現元素標重點」到底好不好玩、值不值得。** 一切設計都為了乾淨、低成本地驗這件事——凍住 `judge` 與成績複雜度、不砸編輯器重工、不讓自動化效果淹沒手放的強調。

## 階段一 — BSR 基底 + 自製 notes(規格已收斂,可動手)

### 形態

**沿用一份既有 BSR 譜面的「基礎資料」(音訊、BPM/timeline、歌名、封面),只把音符換成使用者自製的。** 走 B 路:自製系統把音符生成一份**合成的 v3 難度檔文字**,塞進 `RawMapFiles.difficultyFiles`,基底 Info.dat/音訊沿用該 BSR,續走 `compileChart` 正規化。**不新開繞過 compileChart 的第二條產出路徑**——`compileChart` 仍是唯一正規化點,只做**加法式**擴充(見下「強調如何流過管線」)。

### 音符攜帶什麼

一顆自製音符只帶 **`(時間, 手, 種類)`**:

- **時間**:用**拍(beat)**。生成難度檔把拍填進 `b` 欄,且**沿用基底歌的變速表**(`extractBpmSegments` 從基底難度檔讀 bpmEvents/_BPMChanges → 嵌進生成檔的 bpmEvents),`compileChart` 用它做 beat→秒([compileChart.ts:246](../../src/compile/compileChart.ts#L246))——音符天生落在音樂拍點,**變速歌也對得準**;中段起播的 startSec 用同一條 `buildBeatToSec`,與音符一致。(修正:初版生成檔無 bpmEvents、startSec 用常數 BPM,變速歌會飄掉——2026-07-13 修。)
- **手**:紅/藍 → 左/右(生成難度檔的 `c` 欄)。
- **種類**:`點擊 / 長按 / 強調點擊 / 強調長按`。**鍵不由音符指定**——仍由 `assignKeys` 平衡器指派;「針對性」練某鍵子集靠既有鍵群(`home` / `index-middle`…)。逐鍵 / 逐字母 targeting 留階段二。

### 「強調」= 新表現元素(純表現,`judge` 不動)

強調與 press/hold 正交,是本階段要驗證的核心新元素。**第一階段純表現**:不影響判定窗、combo、成績,`judge` 接縫一行不碰。(鍵盤無力度,強調本質只能是輸出端提示。)

- **視覺**:強調音符**整顆黃色(`0xffd633`,不分手、不放大、不發光)**——一眼可辨;手仍由欄位置傳達。press/hold 皆套。(演進:手色自發光對比不夠 → 發光黃色 → 發光讓鍵字難辨,最終定「不發光黃色」。)
- **音效**:打到強調音符時,**與普通打擊同一顆 tick**,只是**同時送進 echo 匯流排(延遲+回授)長出幾聲迴音**——聲音本體不變,只多迴盪。作法:`playTick(pitch, fancy)`,fancy 時把 tick 的 gain 也接到 echo bus;keydown 用 `outcome.noteIndex` 查 `chart[i].emphasized`。(演進:上行琶音+reverb「方向不對」→ 最終定「原打擊音 + 純迴音」。)
- **來源**:第一階段強調**只靠作者手放**(文字格式的 `!`)。疊放收斂([collapseStacks](../../src/compile/compileChart.ts#L189))遇到群內混合強調時,取**聯集/錨點值**,**不自動升級**成強調(避免全域氾濫、淹沒手放的強調)。

### 強調如何流過管線(compileChart 的加法式擴充)

這是第一階段**唯一**碰到純函式接縫的地方,且為加法式:

- 生成的 v3 音符加自訂欄位(如 `{b, x, y, c, e:1}`,`e`=強調);
- `compileChart` 的 `RawV3Note` + `normalizeV3` 讀 `e`;
- `UnassignedNote` / `Note` type + `assignKeys` 把 `emphasized` 帶到輸出;
- 真實 BSR 音符無 `e` → 預設 false,現有譜面行為完全不受影響。

### 輸入 / 編輯(陽春文字 + 快迭代)

編輯器可陽春到就是一塊 textarea——**重點是快迭代**,不是 UI 美觀。

- **行格式**:一行一顆 `拍 手 種類 [結束拍]`;`!` 尾綴 = 強調;`#` 註解、空白略過。範例:
  ```
  4       L     press
  4.5     R     press!        # 強調點擊
  8       L     hold  12      # 長按,結束拍 12
  8       R     hold! 12      # 強調長按
  ```
- **迴圈(grill 後定案:無試玩鈕、hover 浮現 ▶)**:版面就一塊 textarea;**滑鼠移到某音符行 → 行右緣浮現 ▶,點它 → 從該行的拍「自動演奏」**(免 321 倒數、不手打),看/聽該段效果就走;結算「回選歌」→ 回 textarea、**文字原樣保留**。左鍵/鍵盤純編輯、完全不觸發播放(hover 出鈕不需額外動作);註解/空行不浮現。自動演奏 + 免倒數讓「改一句 → 點浮鈕 → 立刻聽到那段」的迭代最短。
- **錯誤**:語法錯用紅字逐行(含行號)當場提示,清單清空,不崩、不開玩。

### 入口 / 綁定

- **難度選擇畫面**([showDifficultyScreen](../../src/main.ts#L95))多一個「自製音符」選項(如一個假難度)。照常載入一首歌(BSR 代號 / 拖 zip)→ 進難度畫面 → 選「自製音符」→ 開編輯器,配**當下載入那首歌**的音訊/BPM/歌名/封面。
- **零新載入路徑、零新選歌 UI**:載入 BSR 本身就是選歌;issue 16 下載、zip 解壓、Info 解析、音訊解碼整條複用。重新綁 = 再載一次那個 BSR(issue 19 最近清單加速)。

### 試玩:點行起播 + 自動演奏 + 免倒數(grill 後定案)

- **點某一行 → 從該行的拍起播**(無試玩鈕;服務快迭代:改第 32 拍不用從頭)。
- **自動演奏**(複用 `?auto` autoDrive)+ **免 321 倒數**(highway `autoStart`:`beginFromZero` 直接 `freshLaunch`):點一下就看/聽那段,不手打、不等。
- 音訊層**已支援**任意秒數起播([player.ts:69](../../src/audio/player.ts#L69) `play(fromSec)`,`positionSec` 已含偏移)。整譜先 `compileChart`(保留與全曲一致的鍵指派/收斂)再濾 `tSec≥startSec`;前段音符不算 miss(且不記成績)。

### 成績:不記(sandbox)

自製編輯器是 sandbox,試玩只為感受效果,**完全不寫入成績庫**——比照 `?auto` / `?occtest`([main.ts:243-253](../../src/main.ts#L243-L253))。好處:迭代狂重播不洗壞紀錄;**songKey / 成績身分問題整個消失**;中段起播前段 miss 問題一併消失。

### 保存:單份自動草稿

- 編輯中自動把文字存 localStorage,**鍵 = 該首歌 Info.dat 雜湊**(跨來源穩定、與 BSR 代號脫鉤,拖 zip 也適用)。下次載入同一首歌進編輯器,帶回上次文字。
- **只存「一首歌一份草稿」**——不做命名 / 多份 / 譜庫(留階段二)。

## 階段二 — 完整自製 + 進階(待階段一驗證可玩性後再評估)

只有在階段一證明「自製排音符 + 強調」確實提升可玩性、值得加入系統,才展開。屆時重新 grill。已記下的候選:

- **強調影響計分**(階段一刻意不做):讓強調音符打對加更多 / 打錯扣更多。理由值得回頭做——**計分張力會讓玩家更重視強調音符,那份「怕打壞它」的心理本身就是一種更深的強調感受**。要動 `judge` summary + scores 聚合 + 跨鍵群比較。
- **一般譜面的自動強調化時機**:例如「疊放 → 強調」,或其他密集/重拍點自動加重音——套用到所有 BSR 譜(`collapseStacks` 等為共用路徑)。要評估「增色 vs 變吵」,並可能加門檻避免氾濫。等階段一先驗證單純的手放強調,再談自動化。
- **逐鍵 / 逐字母 targeting**:讓音符釘死某鍵(如練「hello」逐字母)。需新開略過 `assignKeys` 的路徑或注入固定鍵,牴觸單一正規化點,是大改動。
- **編輯器進階**:命名 / 多份 / 譜庫、視覺編輯器(節拍格點放)、節奏模式產生器。
- **脫離 Beat Saber 基底**:自帶音訊(節拍器 / tick / 上傳音檔),需替代主時鐘(不再有可靠的 `player.positionSec`)。
- **自製譜正式化 + 記成績**:成績身分(生成難度檔文字雜湊天生不撞原 BSR)、難度 / 鍵群係數對自製譜的意義。
- **分享 / 匯出**:自製譜匯出檔 / 一鍵載入(對應 issue 16 那種體驗)。
