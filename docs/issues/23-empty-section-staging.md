# 23 — 空段落演出

> 來源:使用者需求(2026-07-12,backlog 記錄)。
> **狀態:backlog,方向待 grill。純視覺 / 演出,牽動 issue 06 渲染,不動 compile/judge 計分。**

## 問題 / 動機

Beat Saber 譜面常有「沒有音符」的段落:前奏、間奏 / breakdown、尾奏。本作「歌一律播到底、無血條」(見 CONTEXT「結算」),所以這些空段落必然存在,而目前畫面在這些時刻是**一整段空的高速公路 + 音樂在放、什麼都沒發生**——很死。給空段落一段演出(鏡頭 / 環境 / 燈光律動),能讓沒音符的時間也活著、維持沉浸與節奏感,並自然銜接下一波音符。

## What to build(薄渲染層,只在 highway;不動 compileChart / judge)

偵測「夠長的無音符空檔」,在其間播放一段克制的演出,音符將至時收回、平順回到常態遊玩。

- **偵測**:掃 TypingChart 的 `tSec`(必要時含 hold 尾 `holdEndSec`)找出間隔 ≥ 門檻(如 ≥ 2~3 秒)的空段;純資料計算,放渲染層即可(compile 不需改)。
- **演出候選**(待 grill,可與 issue 24 燈光聯動):鏡頭緩移 / 呼吸、環境光律動、遠景脈動、走廊氛圍變化。**不生假音符、不干擾判定格**。

## 待定(grill 時決)

- **空段門檻**:多長才算「空段」值得演出?太短會頻繁進出、干擾。
- **演出內容**:自成一套,還是複用 issue 24(把譜面燈光事件演在空段)/ issue 20(場景)的元素。
- **進出銜接**:音符將至(提前 flightTime)必須平順收回演出、讓來襲音符清楚可讀,不可搶戲或殘留。
- **開頭 / 結尾**:歌一開始到第一顆音符、最後一顆到歌末,是否也算空段演出。
- **開關**:是否提供關閉並以 **issue 12** 持久化。

## Acceptance criteria(暫,待 grill 收斂)

- [ ] 夠長的無音符段落有演出,不再是死畫面
- [ ] 音符將至時平順收回,來襲音符可讀性不受影響
- [ ] 純視覺:compileChart / judge / 計分不變;playtest 確認進出銜接與不干擾判讀
- [ ] 密集/短空檔不會頻繁誤觸發

## 關聯 / Blocked by

- 依賴 issue 06(高速公路 / 走廊 / 相機)。
- 與 issue 24(燈光轉換)天然聯動:空段正是演譜面燈光的好時機。
- 與 issue 20(場景)共用環境元素;與 issue 14(音符出現提示)互補(一個管有音符、一個管沒音符時)。

## Out of scope

- 有音符段落的演出 / 特效(見 issue 14、21)。
- 在空段生成非譜面的假音符或迷你遊戲(本議只做演出,不改玩法)。
