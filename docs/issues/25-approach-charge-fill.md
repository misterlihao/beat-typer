# 25 — 按下預告:目標格充能填滿(approach charge-fill)

> 來源:使用者需求 + grilling(2026-07-12,設計已收斂,可實作)。
> **狀態:設計定案,待開工。純視覺,只動 highway 渲染層,不碰 compileChart / judge / 音訊。**

## 問題 / 動機

音符飛來時,玩家要判讀 3D 深度才知道「何時該按」。在判定平面的**目標格**上疊一個「充能填滿」預告:抵達前兩拍,該格中央長出半透白片、恰好在**打擊瞬間(pHead=1)填滿**,再自然交棒給既有打擊發光。等於在平地上給一個不必判讀深度的計時讀數,和天上的方塊互補。

## What to build(薄渲染層,只在 highway)

每個「要按下」的音符,在其抵達前的**提前窗**內,於目標格畫一片白色方片,由中心 `scale 0→1` 長大、`opacity 0→~0.35` 淡入,**在 pHead=1 剛好填滿並與鍵格邊界完全重合**;判定一到即撤掉白片,由既有 `activateCell` 在同位置點出判定色發光(白→亮色的自然交棒)。

### 定案規格(grilling 收斂,逐點)

- **提前窗 = 兩拍**(`CHARGE_LEAD_BEATS=2`;實跑手感:一拍偏短,兩拍較看得清):`lead = min(2 × 60/bpm, flightTime)`。跟著音樂呼吸(快歌短、慢歌長),夾住不超過飛行時間。
  - **bpm 缺失 / 為 0 / 非法 → 拍長退回估計 `0.5s`(120bpm),窗 = `0.5 × 2 = 1.0s`(仍夾 flightTime)**;不可悄悄退化成整段 flightTime(= 滿螢幕,已否決)。
  - 全曲單一 BPM,**忽略曲中變速**(視覺預告,不參與判定,近似可接受)。需把 `beatSec`(或 bpm)傳進 `startHighway`(現簽章沒有;`main.ts` 手上的 `info.bpm` 可傳)。
- **對齊**:充能進度 `p` 錨定音符飛行 `arrival = tSec + offsetSec`,`p=1` 即 pHead=1。跟著現有 `flightTime` / `offsetSec`,不另立時鐘(offset 滑桿一動,白片與方塊同步位移)。
- **形狀**:和鍵格同形、同位置的**白色方片**(複製 `cellGeo`),由中心 scale。填滿四角、與格線邊界對齊(不用圓——填不滿長方格、交棒會形狀跳變)。
- **獨立白片,不共用打擊發光那片格**:避免每幀在「充能」與「閃光/持續亮/Miss 紅」之間仲裁、以及 scale 重置的糾纏。兩套系統互不干擾:判定到 → 白片歸零、閃光照常跑。材質/幾何可共用,開銷極小(10×3 或一個池)。
- **線性、無 easing**:scale 與 opacity 都隨 `p` 線性上升。等速長大 = 恆定填滿速率,本身就是計時線索。
- **滿格 opacity ≈ 0.35**(半透白):明顯低於打擊發光峰值(金 0.95 / 綠 0.85 / 紅 0.7 / 中性 0.32),判定一到「白→亮色」清楚變亮,交棒有層次。
- **純白、左右手一樣**:白片是中性的計時預告,和判定色 / 手色(紅左藍右)分工清楚。

### 三種結局的收尾(白片長到滿 → 判定一到即被判定色取代)

- **命中(金/綠)**:白片立刻撤、同位點出判定色滿格發光 →「白片點燃成色」。
- **自動 Miss**(遲到窗過了沒打):白片撤、點出紅色 Miss 發光 →「滿了、等不到、變紅」。
- **提前命中**(pHead=1 前的提早窗內打中):白片未滿(如 80%),命中時仍立刻撤、點滿格判定色。接受一下「小白→滿色」放大跳(提早命中少,那一下反而像達成獎勵)。
- **滿了、判定未出的短暫空檔**:維持滿白不動,順便暗示「還能補救」。

### 長按

- **頭:與 press 完全一樣**——頭抵達前兩拍充能、滿格交棒給頭部命中發光。
- **尾:不做充能**——按住期間目標格本就持續發光(`heldNotes`),尾巴鎖定另有金脈衝 + 完成音;疊白片只會打架變吵。
- 收斂:**充能只發生在「要按下」的一刻(所有 press + 長按的頭)**。

### 實作提示

- per-cell 驅動:每幀對每格找「該鍵最近一顆、未判定、且已進入提前窗」的音符算 `p`,驅動該格白片;沿用既有 per-frame 更新,**不開 per-note timer**。
- 白片 z 位置與鍵格幾乎重合(如 `PLANE_Z - 0.021`),滿格時和發光格對齊,交棒無縫。

## Acceptance criteria

- [ ] 每個 press / 長按頭在抵達前兩拍(`min(2×60/bpm, flightTime)`)於目標格長出白片,pHead=1 剛好填滿
- [ ] bpm 非法時拍長退回 0.5s(窗 1.0s),不會變成整段 flightTime
- [ ] 命中 → 白片撤 + 判定色滿格發光;自動 Miss → 白片撤 + 紅發光;白→亮色可見地變亮(交棒自然)
- [ ] 提前命中不殘留白片;滿格未判定時維持滿白
- [ ] 長按只有頭充能,尾不充能;按住持續發光不受白片干擾
- [ ] offset / flightTime 滑桿一動,白片與方塊、判定同步
- [ ] 純視覺:compileChart / judge / 音訊不變;密集段不掉幀、不殘留(無 per-note timer 洩漏)
- [ ] playtest-highway 實跑確認:填滿對齊打擊瞬間、白片不被誤讀為判定結果、交棒無縫

## 關聯 / Blocked by

- 依賴 issue 06(高速公路 `positionNotes` / `cells` / `activateCell` / `pHead`)。
- **與 [issue 14](14-note-spawn-rhythm-cue.md) 同屬「目標格預告」家族但不同時機**:14 是音符**首現(FAR_Z)當下**的短脈動,本 issue 是**抵達前兩拍**的持續充能。**若本 issue 先落地,14 應避免再走候選 B(目標格預閃)**,以免同一格雙重預告互相混淆;14 改走 A(音符彈入)或 C(遠端 spawn 脈衝)較不撞。
- 可選:提供開關並以 **issue 12** 持久化(嫌花的玩家可關)。
- 白片為純白、且判定一到即讓位,天生滿足「不可誤讀為判定色」的區隔約束。

## Out of scope

- 長按尾部充能、背景律動 / 小節線拍格(另議,見 issue 14 Out of scope)。
- 依手色(紅/藍)上色的充能(本 issue 定為純白)。
- 打擊粒子 / 螢幕震動等更重特效。
