# 19 — 賽前 / 開始畫面

> 來源:grilling issue 09 衍生(2026-07-12)。
> **狀態:設計待 grill。依賴 issue 18(成績持久化)。**

## 問題 / 動機

目前選定難度後**直接進高速公路**,「開始」只是控制列一顆 `▶ 開始` 鈕(`highway.ts` 的 `.bt-start`),按下即從 0 播。缺一個賽前的「準備」時刻:確認要玩哪首/哪難度、看**最佳成績當目標**、再開始。有了 issue 18 存下的成績,這個畫面才有內容可秀。

## What to build(待 grill 定形態)

在難度選定 → 高速公路之間(或取代現有 `▶ 開始` 鈕)給一個賽前畫面/卡片:歌名 + 封面 + 難度 + 鍵群 + **最佳成績(評級/準確率/最大 combo,來自 issue 18)** + 「開始」。

### 待定(grill 時決)

- 形態:獨立賽前頁,還是高速公路進場前的一張覆蓋卡(和暫停/結算覆蓋層一致的路數)?
- 與現有 `▶ 開始` 鈕 / issue 13 狀態機怎麼銜接(idle 狀態即賽前?)。
- 無成績(首次玩)時顯示什麼。
- 和 issue 09 結算的「回選歌 / 重新開始」導覽如何串成完整循環(選歌 → 賽前 → 玩 → 結算 → 賽前/選歌)。

## Acceptance criteria(暫,待 grill 收斂)

- [ ] 進歌前有賽前畫面,顯示歌/難度/鍵群與最佳成績(有成績時)
- [ ] 「開始」接上既有 `beginFromZero`
- [ ] 首次玩(無成績)畫面合理不空洞
- [ ] 與 issue 09 導覽串成閉環
