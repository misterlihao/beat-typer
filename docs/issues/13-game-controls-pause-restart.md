# 13 — 遊戲控制(暫停 / 繼續 / 重新開始)

> 來源:使用者需求——需要暫停、重開之類的基本遊戲控制。

## 問題

現行高速公路只有「開始」鈕(`.bt-start`),按下即從 0 播;歌自然結束後 `onEnded` 暫停停在結尾。**遊玩中無法暫停/繼續**,也**沒有乾淨的重新開始**(start 會從 0 重播音訊,但不重置 combo / judge / 發光 / 音符可見性)。AudioPlayer 已備 `pause()` / `play(fromSec)` / `stop()` 原語,缺的是上層控制與狀態重置。

## What to build

在高速公路(薄渲染/輸入層,不動純函式接縫)加三個控制:

- **暫停 / 繼續**:遊玩中暫停 → 停音訊(`player.pause()`)、停動畫迴圈(`cancelAnimationFrame`)、輸入不判定;繼續 → `player.play(原位)` 續播、迴圈重啟,combo / 判定狀態延續。時鐘=`positionSec` 凍結,故暫停期間不會有時間流逝造成的假 Miss。
- **重新開始**:當前譜 + 難度從 0 重來——重建 `Judger`、清空 combo / 判定閃字 / 格子發光、還原所有音符可見性,`player.play(0)`。
- **控制鍵**:須避開 30 個遊戲鍵(QWERTYUIOP ASDFGHJKL; ZXCVBNM,./)。用**非遊戲鍵**:建議 `Escape` 或 `Space` = 暫停/繼續(`preventDefault` 防捲動);重新開始用畫面按鈕(避免綁字母鍵)。
- **暫停 UI**:明確的「暫停中」覆蓋層 + 繼續 / 重新開始按鈕。

## Acceptance criteria

- [ ] 遊玩中可暫停:音訊停、音符凍結、按鍵不判定;繼續從原位續播且 combo/判定延續
- [ ] 暫停→繼續不產生因暫停造成的假 Miss(凍結 positionSec,expiry 不誤觸發)
- [ ] 重新開始:當前譜/難度從 0,combo / 判定 / 發光 / 音符可見性全重置
- [ ] 控制鍵不與任何遊戲鍵衝突;暫停時顯示明確 UI,可用鍵盤與按鈕操作
- [ ] 歌曲自然結束後仍可重新開始(承接現行 onEnded)
- [ ] 渲染/音訊維持薄層;compileChart / judge 純函式不變
- [ ] 實跑驗證:暫停/繼續/重開在瀏覽器行為正確(playtest-highway)

## 設計定案(grill-with-docs,2026-07-11)

- **暫停/繼續鍵**:`Space` 與 `Escape` **都**切換暫停/繼續(皆非遊戲鍵),按下 `preventDefault`(防捲動/防誤觸隱藏的開始鈕)。
  僅在「遊玩中/暫停中」作用;idle / ended 忽略。開始維持畫面按鈕(不讓 Space 兼開始,語意單純)。
- **暫停覆蓋層**:半透明暗幕 + 「暫停中」+〔繼續〕〔重新開始〕兩鈕(可點、可 Tab+Enter)。
  「重新開始」入口只兩處:暫停覆蓋層、歌曲結束畫面(想中途重來 → 先 Space 暫停 → 重新開始;不在遊玩畫面常駐擋視線)。
- **暫停中可調滑桿**(使用者追加):底部控制列(飛行時間/offset/音量)z-index 疊在覆蓋層之上、暫停時固定顯示,
  讓玩家就地校準後再繼續(調整即時套用 + 持久化);繼續/重開時還原遊玩中的滑鼠靠近才顯。
- **失焦自動暫停**:監聽 `document.visibilitychange`,一 hidden 就走暫停路徑。堵掉「背景分頁凍結 rAF 但音訊續播 →
  回來時 `expiry(now)` 把整段一次判 Miss、combo 全斷」的 correctness bug(複用同套暫停邏輯)。
- **重新開始 = 可重用 reset 函式**(startHighway 內):重建 Judger、清 combo/閃字/格子發光、還原音符可見性、`play(0)`、
  必要時重啟 rAF、收掉覆蓋層。issue 09 重玩鈕日後直接呼叫,不寫第二份。
- **測試接縫**:控制全在 highway 薄層、無純接縫 → **僅靠 playtest-highway 實跑驗**(暫停/繼續/重開/失焦),不寫 vitest。

## Blocked by / 關聯

## Out of scope

- 任意位置 seek / 拖曳進度(只做暫停續播與從 0 重開)。
- 變速播放。
- 跨 session 存檔續玩(關掉再開從中斷處繼續)。
