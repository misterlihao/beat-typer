# 11 — 重新設計映射以平衡打字練習價值(設計型)

> 來源:使用者觀察——Beat Saber 譜面有位置偏重,直接映射成打字練習時 finger load 不均、家排被餓死。
> **狀態:設計已收斂(2026-07-10 grill-with-docs)。決策見 docs/adr/0008(取代 0002)與 CONTEXT「鍵指派/教學權重/可玩性」。以下「設計收斂」為權威規格,舊的候選方向 A–E 僅存為推理紀錄。**

## 設計收斂(權威規格)

grill-with-docs 逐一裁決的七個分叉:

1. **主目標**:衝突時**打字價值 > Beat Saber 空間手感**。
2. **determinism**:**單場內**(同譜+難度可 replay;跨譜各自平衡,不保證同音符對同鍵)。
3. **機制**:**完全解耦 + 自由平衡器**——手仍由顏色決定,手指/排(最終鍵)不由欄/列決定,由鍵指派選。位置(欄/列)對鍵不再有作用。
4. **高速公路**:道與高度改依**被指派鍵**的鍵盤位置渲染;鍵盤形保留、渲染層幾乎不返工。
5. **平衡目標**:**家排加權教學分布**(家排>上>下、食指/中指>無名/小指);權重皆正 → 覆蓋自然浮現;權重封頂峰值。
6. **可玩性硬底線**:同手、緊湊時間窗(初值 ~120ms)內不得指派同一手指;平衡在可玩集合內做;per-note 指派。
7. **手均衡 = 顯式非目標**:顏色→手不變,左右手總量 = 譜紅/藍比例,只在各手內攤平。

**實作**:鍵指派為 `compileChart` 內、正規化+疊放收斂之後、輸出之前的純函式 pass,無 RNG,可 fixtures 測。疊放**收斂**保留(節奏),但 ADR 0006 的「收斂後落內側鍵」語義取消——內側鍵降為鍵池 6 個普通鍵。PRD 全格映射表降為「鍵池與鍵盤位置參照」。

## 問題(已量化)

以 `5277c (Masquerade)` ExpertPlus(747 顆)經**現行映射**(顏色→手、欄→手指、列→排)後的按鍵分布:

- 最熱 4 鍵 `,` `C` `;` `A` ≈ **49%**;打字最核心的家排食指/中指 `F D J K` 各僅 **0.3–0.5%**。
- **9 鍵整場未出現**:`R T Y U G H B N M`。
- 列偏重:下排 **52%** / 家排 25% / 上排 23%(對觸打本末倒置)。

Beat Saber 的空間分布(偏下排、偏中間欄、雙手大致均衡)拿來當打字練習,結果是**指負荷極不均、家排練不到、單場覆蓋不全**。

## 這次改動要對付的核心取捨

1. **CONTEXT「映射」的 determinism**:現在「同一 (顏色,欄,列) 永遠同一鍵」用來建立肌肉記憶。均衡化(尤其跨譜自適應)可能弱化這點。
2. **ADR 0002 高速公路空間對應**:現在「道↔欄、列↔排」,高速公路長得像鍵盤。若鍵脫離空間位置,視覺一致性會斷——除非高速公路跟著重排。
3. **打字教學價值**(新目標):指負荷均衡、家排優先、盡量全覆蓋。

三者互相拉扯,這正是本 issue 要在 grilling 裡裁決的主軸。

## Acceptance criteria(依設計收斂定案)— 全數達成 ✅(2026-07-11)

- [x] `5277c (Masquerade)` ExpertPlus 經新鍵指派後:峰值 13%→**7.2%(F)**、家排 F D J K 各 0.3–0.5%→**各 7.2%**、下排 52%→**16%**
- [x] 各鍵出現量朝「教學權重」趨近:家排 > 上排 > 下排、食指/中指 > 無名/小指(實測分布單調符合)
- [x] 覆蓋顯著改善:Masquerade **30/30 鍵全中**(舊有 9 鍵未出現)
- [x] 可玩性硬底線成立:同手緊湊時間窗內無同指連續指派(keyAssignment.test 斷言)
- [x] 顏色→手 不變:左右手 49.9/50.1;手均衡非目標,只各手內攤平(fixtures 斷言)
- [x] determinism 單場內:同譜兩次編譯完全相同(fixtures 斷言);無 RNG
- [x] 疊放收斂仍成立;收斂後音符進入鍵指派(不再固定落內側鍵,fixtures 斷言)
- [x] `compileChart` 仍為純函式主接縫;鍵指派為可決定性純函式(keyAssignment.ts)、fixtures 測
- [x] 高速公路:音符依被指派鍵飛入正確道/高度(瀏覽器實跑內建範例全連 maxCombo 8)

**實作**:src/compile/keyAssignment.ts(鍵池+教學權重+可玩性+D'Hondt)。測試 67 全過、tsc/build 乾淨。

## 候選方向(非決議,供 grilling 展開)

- **A. 重排靜態映射表**:仍 (顏色,欄,列)→鍵 可決定,但改選「哪個位置對到哪個鍵」以把最熱位置放到家排。保留 determinism 與空間道;但無法讓「原譜沒用到的位置」被練到(排列不改分布形狀,只改標籤)。
- **B. 逐譜自適應排列**:依該譜實際位置分布選一個排列使負荷攤平。場內一致、跨譜不一致。
- **C. 鍵與空間解耦**:高速公路仍照 Beat Saber 空間、但鍵標籤另配以求均衡(道≠鍵位,需解決可讀性/直覺)。
- **D. 隨時間輪替配置**:映射隨進度輪轉以覆蓋更多鍵。
- **E. 不改映射,加練習模式**:接受偏重,另提供「全鍵均衡」模式或字表。
