# 20 — 遊戲場景 / 環境美術(軌道之類的)

> 來源:使用者需求(2026-07-12,backlog 記錄)。
> **狀態:backlog,方向待 grill。純視覺、牽動 issue 06 渲染,不動 compile/judge。**

## 問題 / 動機

現在的高速公路是「懸空的格線 + 深色底」:判定平面一張 10×3 鍵盤格線、音符沿走廊飛來,除此之外沒有任何場景。缺少「這是一個地方」的空間感與沉浸感。加入環境美術(軌道 / 導軌 / 地平線 / 遠景律動)能讓畫面像一條真正的跑道,而非漂浮的網格,也更符合節奏遊戲的舞台感。

## What to build(薄渲染層,只在 highway;不動 compileChart / judge)

沿音符飛行走廊佈置環境元素,把「走廊」實體化。候選(待 grill 取捨):

- **軌道 / 導軌**:沿走廊兩側(或每道之間)的線條 / 帶狀幾何,強化透視收斂與速度感,順帶暗示 10 道的分界。
- **地面 / 天空 / 地平線**:給走廊一個「站立面」與遠方消失點的背景,取代純黑。
- **遠景律動(可選)**:呼應音樂 / BPM 的低調背景動態(不搶戲)。

## 待定(grill 時決)

- **份量**:極簡(只加兩條導軌 + 地平線)還是完整環境(地面材質、天空盒、氛圍光)。先求不搶戲的最小版?
- **與既有幾何的關係**:走廊已是向上彎曲的曲線(`liftAt`),軌道要不要跟著彎?道間距 `LANE_SPACING`、抬升 `LIFT_MAX` 是否共用真實來源。
- **不干擾判讀**:場景不可搶走音符 / 判定格 / 字母的注意力;顏色與左右手紅藍、判定金綠紅要能區隔。
- **效能**:靜態幾何一次建立、避免每幀重算;行動裝置可接受。
- **開關 / 品質**:是否提供開關或簡易品質分級,並以 **issue 12** 持久化。

## Acceptance criteria(暫,待 grill 收斂)

- [ ] 高速公路有可辨識的場景 / 軌道,空間感明顯優於純格線
- [ ] 不干擾音符可讀性與判定回饋辨識(顏色 / 亮度 / 位置有區隔)
- [ ] 純視覺:compileChart / judge / 音訊不變;playtest 確認不掉幀、不殘留
- [ ] 與走廊既有幾何(彎曲 / 道距 / 抬升)一致,不各立常數
