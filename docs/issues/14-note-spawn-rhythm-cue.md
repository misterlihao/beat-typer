# 14 — 音符出現的節奏提示動畫(spawn cue)

> 來源:使用者需求——節奏遊戲常見:音符出現時來個簡單動畫,提供額外的節奏提示。
> **狀態:設計方向待定(候選見下),牽動 issue 06 渲染,不動 compile/judge。**

## 問題 / 動機

現在音符只是「無聲無息地」在遠端(FAR_Z)現身、飛向判定平面(highway.ts:`positionNotes`,`pHead` 從負跨 0 即出現)。缺少一個在**音符出現當下**的視覺脈動。加一個出現時的簡單動畫,能:

- 給玩家**額外的節奏提示**——出現事件發生在 `tSec + offset − flightTime`(固定提前量),與命中節奏同拍、只是提早,等於把節奏在畫面上多敲一次。
- 讓「新音符來了」更好察覺,尤其密集段。

## What to build(薄渲染層,只在 highway;不動 compileChart / judge)

在每顆音符**第一次變可見**時觸發一個簡短(~100–200ms)、克制的出現動畫。天然掛鉤:`positionNotes` 裡 `pHead` 由 <0 跨到 ≥0(hold 同理),per-note 記一個 `spawned` 旗標只觸發一次。

### 候選方向(擇一或組合,待 grill / 實作時定案)

- **A. 音符本身彈入(推薦)**:出現瞬間音符做 scale(小→正常)或 alpha(淡入)+ 輕微 overshoot。自足在音符 mesh 上、與任何既有回饋零混淆、最便宜。直接回應「音符出現時有一個簡單動畫」。
- **B. 目標格預閃(「亮一下軌道」)**:出現時在判定平面**該鍵的格子**輕閃一下(復用 `activateCell`,但**須用與判定色(金/綠/紅/中性)明確不同的顏色**,例如暗青),等於預告落點位置+時機。提示最強,但與 hit 發光同一組格子 → 混淆風險最高,需嚴格區隔顏色/亮度。
- **C. 遠端 spawn 特效**:出現點(FAR_Z)來個小脈衝/光環一閃。空間上與 hit 回饋完全分離、無混淆;但在遠端小、可讀性較弱。

### 設計約束

- **與判定回饋區隔**:不可讓玩家把 spawn 提示誤讀成 Perfect/Good/Miss。顏色、位置、時機至少一項要明顯不同。
- **克制**:短、淡、不搶戲;密集段不能糊成一片或拖累效能(復用既有 per-frame 淡出機制,別每顆各開 timer)。
- **與 offset/flightTime 一致**:出現時機跟著現有 `flightTime` 與 `offsetSec`,不另立時鐘。

## Acceptance criteria

- [ ] 每顆音符出現(首次可見)當下觸發一次簡短動畫,只觸發一次、不重複
- [ ] 動畫與判定發光/閃字明顯可區分(不被誤讀為判定結果)
- [ ] 密集譜段不掉幀、不殘留(沿用既有淡出、無 per-note timer 洩漏)
- [ ] 純視覺:compileChart / judge / 音訊不變;實跑(playtest-highway)確認出現脈動與節奏同步、不干擾判讀
- [ ] (若做 B)spawn 格閃色與 CELL_COLOR 的判定四色不撞

- 可選:提供開關,並用 **issue 12** 持久化(有些玩家嫌花)。

## Out of scope

- 與音符無關的獨立節拍脈衝 / 背景律動(本 issue 只做「音符出現」觸發)。
- 小節線 / 拍線 tempo 網格(另議)。
- 打擊粒子、螢幕震動等更重的特效。
