# 15 — 可選擇要訓練的按鍵群

> 來源:使用者需求——想針對特定按鍵做重點練習(如只練家排、只練食指/中指、只練左手),而非每次都吃整張鍵盤的均衡分布。

## 問題

issue 11 定案的鍵指派會朝「教學權重」把整池 30 鍵攤平(家排>上>下、食指/中指>無名/小指)。這對「全面練習」很好,但無法服務**針對性訓練**的需求:

- 初學者想先只練**家排 8 鍵**(`A S D F J K L ;`),不想被上/下排干擾。
- 想單獨鍛鍊**弱指**(無名/小指),或反過來只練**強指**(食指/中指)暖手。
- 想只練**單手**(如慣用手受傷、或分手練習)。

目前鍵池(`buildPool`)寫死為某手全部 15 鍵,玩家無從縮小訓練範圍。這是 issue 11 當時列為候選方向 E(「不改映射,加練習模式」)、當時未做的那條路,現正式切成獨立 issue。

## What to build

一個**鍵群選擇**,把鍵指派的可用鍵池限縮到玩家選定的子集,其餘機制(教學權重攤平、可玩性硬底線、D'Hondt、determinism)原封不動,只是在更小的池上運作。

- **接縫**:鍵群為 `compileChart` 的 config 輸入(`CompileConfig` 新增欄位,如 `keyGroup?`),向下傳給 `assignKeys` 縮小 `buildPool` 的產出。純函式性質不變——同一 (譜+難度+鍵群) 永得同一組指派(單場內 determinism 延伸涵蓋鍵群維度)。
- **預設鍵群**(至少):全鍵(現況,預設)、家排、家排+上排、食指/中指、只左手、只右手。是否開放任意勾選單鍵可後續再議,先給預設組。
- **UI**:載入層/開場選單提供鍵群挑選(薄層,不放邏輯);與難度挑選同層。
- **持久化**:鍵群屬跨場玩家偏好,沿用 issue 12 的設定層存 localStorage(若 12 已完成則接上;未完成則本 issue 至少讓選擇在單次 session 生效)。

## 這次改動要對付的核心取捨

1. **顏色→手 vs 鍵群衝突(最尖)**:手仍由顏色決定(CONTEXT 唯一保留的位置語義)。若玩家選「只左手」,但譜有藍(右手)音符——該手在選定鍵群內**沒有任何鍵**。必須裁決:
   - (A) 鍵群只在「該手仍有鍵」時作用,某手被清空時該手音符**回退整池**(顏色語義優先);或
   - (B) 鍵群可**凌駕顏色**,把藍音符也塞進左手鍵池(等於放棄顏色→手,單手練習時可接受);或
   - (C) 單手鍵群時**濾掉**另一手的音符(改變 TypingChart 音符數與音同步預期)。
   三者對 CONTEXT「手」定義與 judge 準確率分母影響不同,需明確定案(建議進 grilling)。

2. **可玩性硬底線可能不可滿足**:鍵群縮到極小(如單手只留 2 鍵)時,同手緊湊時間窗內勢必得重用同一手指。既有 `assignKeys` 已有「全被佔→放寬回整池」的降級,但那會回退到**整池**而非**選定鍵群**,等於破壞鍵群約束。需決定:放寬時只在鍵群內放寬(容忍同指連打)還是暫時借用群外鍵。

3. **教學權重在子集內的意義**:攤平目標在縮小池上仍成立(D'Hondt 對子集照樣運作),但「家排>上>下」的相對權重在「只家排」時退化為純輪替——確認這是可接受的(應是)。

## Acceptance criteria

- [ ] 選定鍵群後,`compileChart` 產出的 TypingChart 只用該鍵群內的鍵(fixtures 斷言:各預設群的鍵集正確)
- [ ] 全鍵(預設)行為與現況完全一致(不回歸 issue 11 的分布/覆蓋成果)
- [ ] determinism:同一 (譜+難度+鍵群) 兩次編譯完全相同;無 RNG(fixtures 斷言)
- [ ] 顏色→手與鍵群衝突的裁決(取捨 1)有明確且測到的行為(依定案的 A/B/C)
- [ ] 可玩性:鍵群縮小時同手同指最小間隔的處理有定義且測到(取捨 2)
- [ ] 鍵群為 `compileChart` config 輸入;keyAssignment 仍為純函式、fixtures 測;無新 I/O 相依
- [ ] UI 可挑鍵群(薄層);選擇在瀏覽器實跑時生效(playtest-highway 驗:選家排→只有家排格會亮)

## Blocked by / 關聯

- 直接延伸 issue 11(候選方向 E)與 docs/adr/0008;鍵池與教學權重在 keyAssignment.ts。
- 與 issue 12(設定持久化)協作:鍵群為跨場偏好,理想上經 12 的設定層持久化。建議 12 先或同步。
- 觸及 CONTEXT「鍵指派 / 鍵池 / 手 / 可玩性」;若採取捨 1-B/1-C(鍵群凌駕顏色或濾音符),需回寫 CONTEXT 並可能開新 ADR。
- 不阻擋 09/13/14。

## Out of scope

- 逐字母自訂字表 / 單字練習(這是「鍵群」不是「詞庫」)。
- 依表現自動調整鍵群(自適應難度)——本 issue 只做玩家手動選。
- 改動判定/計分規則本身(judge 不變,只是消費更小鍵集的 TypingChart)。
