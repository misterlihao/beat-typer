# 21 — 雙手音符特效(同拍紅+藍)

> 來源:使用者需求(2026-07-12,backlog 記錄)。
> **狀態:backlog,方向待 grill。純視覺 / 音效回饋,牽動 issue 06/07 渲染與回饋,不動 judge 計分。**

## 問題 / 動機

TypingChart 允許**跨手同時**——紅 + 藍同一拍共用時間(見 CONTEXT「TypingChart」)。這種「雙手音符」在演奏上是一個強調時刻(左右手同時出手),但目前兩顆音符各自獨立飛來、各自獨立回饋,沒有任何「這是一組」的視覺語言。給同拍雙手音符一個特效,能凸顯這些節奏重點、增加打擊爽感。

## What to build(薄層;compileChart / judge 不變)

辨識「同時的左右手音符」並給予強化的視覺(可能加音效)回饋。兩個時機點可掛鉤:

- **來襲時**:兩顆同拍音符之間的連結感(如連線 / 共同光帶 / 同步脈動),讓玩家預見「這拍要雙手一起」。
- **命中時**:兩顆都在窗內命中時,額外的合擊特效(比單顆更大的閃 / 光爆 / 專屬音),獎勵同步。

## 待定(grill 時決)

- **「同時」的定義**:嚴格同一 `tSec`?還是相距某個極小窗內即算一組?(compile 已把同手疊放收斂,但跨手同拍是保留的。)辨識放渲染層即可,不需動 compile。
- **特效形態**:連線 / 光帶 / 合擊閃 / 專屬音,擇一或組合。
- **與判定回饋區隔**:不可讓合擊特效被誤讀成某個判定等級;沿用 issue 14 的區隔紀律(顏色 / 位置 / 時機至少一項不同)。
- **命中條件**:兩顆都 Perfect?只要都命中(含 Good)?其中一顆 Miss 時的降級表現。
- **克制與效能**:密集雙手段不能糊成一片或掉幀;沿用既有 per-frame 淡出、不每組開 timer。

## Acceptance criteria(暫,待 grill 收斂)

- [ ] 同拍左右手音符被辨識為一組,並有可見的「成組」語言(來襲 and/or 命中)
- [ ] 特效不被誤讀為判定結果;與判定發光 / 閃字明顯區分
- [ ] 純呈現:compileChart / judge 計分不變(雙手仍是兩個判定單位)
- [ ] 密集段不掉幀、不殘留;playtest 確認同步感與不干擾判讀

## 關聯 / Blocked by

- 依賴 issue 06(高速公路 / 音符視覺)與 issue 07(判定回饋 / `flash` / `activateCell`)。
- 與 issue 14(音符出現提示)、issue 20(場景)同屬視覺打磨;issue 14 已把「打擊粒子 / 螢幕震動」列為其 out-of-scope,重特效歸此議。

## Out of scope

- 一般(非同拍)單顆音符的打擊特效升級(另議)。
- 改動計分 / combo 規則(雙手仍各記一個判定)。
