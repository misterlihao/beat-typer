# 映射改為「單場內打字平衡的鍵指派」,取代固定空間順序映射

當「Beat Saber 空間手感」與「觸控打字練習價值」正面衝突時,以**打字價值為主**。因此廢除 ADR 0002 的固定 `(顏色,欄,列)→鍵` 空間順序映射,改為:**手仍由顏色決定,但手指與排(最終的鍵)由一個「鍵指派」在單場內為打字練習平衡而選**,不再由欄/列決定。

## 為何記錄

這直接推翻 ADR 0002(刻意採空間順序、讓高速公路像鍵盤),是硬轉向、代價明確,未來讀者會問「為何不照 Beat Saber 位置映射了」。留痕。

## 觸發問題(已量化)

`5277c (Masquerade)` ExpertPlus(747 顆)經舊映射後:最熱 4 鍵 `, C ; A ≈ 49%`(單鍵峰值 ~13%)、家排食指/中指 `F D J K` 各 0.3–0.5%、**9 鍵整場未出現**、下排佔 52%。作為打字練習,指負荷極不均、家排被餓死、覆蓋不全。

## 關鍵推理

- **數學事實**:舊映射是 `(顏色,欄,列)→鍵` 的固定雙射,每鍵負荷 = 該格在譜中的出現頻率。重排這張表只是換標籤,頻率的多重集合不變——峰值搬得動位置但值不變、未用格永遠未用。**壓平峰值 / 補覆蓋在固定雙射下數學上不可能**,必須讓「同一輸入在不同時候對到不同鍵」,即打破雙射。
- 打破雙射與「跨譜永久 determinism」不能並存,取**單場內** determinism:同一譜+難度可 replay、可建立該譜的肌肉記憶;不同譜各自平衡。
- 打字為主時玩家應**讀字母**(觸打回想訓練),故「道能預測鍵」非必要;既然自由,索性**完全解耦**,由平衡器直接優化分布,而非在位置映射上打補丁。

## 決策細節

- **平衡目標 = 家排加權教學分布**:目標佔比 家排 > 上排 > 下排、食指/中指 > 無名/小指。權重皆正 → 覆蓋自然浮現(best-effort,短譜可能漏鍵,不為強制覆蓋扭曲平衡);權重封頂各鍵峰值。
- **可玩性為硬底線**:同一手、時間相鄰過近(緊湊時間窗,初值 ~120ms)的音符不得指派同一手指;平衡只在滿足底線的選項中進行。per-note 指派(正是這自由讓避開同指成為可能)。
- **顏色→手 保持不變(顯式非目標:手均衡)**:左右手總負荷 = 譜的紅/藍比例,不由指派調整;只在各手內攤平。保住 PRD「雙手分工」與高速公路左右=手。
- **實作位置**:鍵指派是 `compileChart` 內、正規化與疊放收斂之後、輸出之前的一個純函式 pass,無 RNG、可決定性,可用手寫 fixtures 測(主接縫不變)。

## 連帶影響

- **ADR 0002**:superseded。空間順序映射的推理不再適用。
- **ADR 0006(疊放收斂→內側鍵)**:疊放**收斂**本身(節奏/可玩性)保留;但「收斂後那顆落內側鍵」的語義取消——收斂後的音符與其他音符一樣交給鍵指派,內側鍵 T/G/B·Y/H/N 降為鍵池中 6 個普通鍵(各帶教學權重)。
- **高速公路(issue 06)**:渲染幾乎不返工——道與高度改依「被指派鍵」的鍵盤位置(原本依原譜欄/列)。鍵盤形與手指預告保留,spatial 手感降為讀性輔助。
- **PRD 映射表**:那張固定全格表不再是映射真相來源,降為「鍵池與鍵盤位置參照」;映射真相移至本 ADR 與 CONTEXT「鍵指派」。
