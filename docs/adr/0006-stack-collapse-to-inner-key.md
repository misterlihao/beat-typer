# 同手疊放收斂成單一內側鍵,不做 burst 展開

**Status:** accepted

同一手、相距 < 1/8 beat 的多顆音符(疊放)**整組收斂成一顆內側鍵音符**(左 T/G/B、右 Y/H/N,依錨點那顆的列取上/中/下),而非展開成多鍵連打。相距 ≥ 1/8 beat 的連打則各自保留正常映射。此決策取代 PRD user story 14/15 原本設想的「同拍展開為 burst + 額外音符落內側鍵」。

## 為何記錄

PRD 原本明訂同拍多音符展開為「極短固定間隔的連打」、額外音符再落到內側鍵。實作時反轉了這個設計:疊放**收斂為一顆**、內側鍵當「強調」而非「額外鍵」。一個看過 PRD 的人會預期看到 burst 展開邏輯,卻在程式裡找不到,值得留痕。

## 關鍵推理

- Beat Saber 中同手同一瞬間的多方塊本來就是**一次揮砍的強調**,不是要玩家連砍多次;展開成連打是對原譜的失真。
- 內側鍵 T/G/B(或 Y/H/N)是**同一根食指的三個列**,若一次要按多顆內側鍵,物理上得靠同指連擊,高 BPM 下敲不到、必掉 combo。收斂成一顆就沒有這個問題。
- 「落在內側鍵」本身已足以表達強調感,不需要再額外多生一個鍵。
- 因此攤平時間全程在 beat 空間計算、只在最後換一次秒;不再需要「burst 間隔」這種秒數常數,單位保持一致。

## 被否決的替代方案

- **PRD 原案(同拍展開 burst + 額外落內側鍵,額外鍵在時間上錯開)**:錯開量的目的是「同一根食指的連擊速度」,本質是毫秒生理限制,會逼著在 beat 管線裡混入秒數常數;且高 BPM 下同指連擊敲不到。取捨上選擇忠實原譜揮砍 + 單位一致,放棄「保留打字密度」。
- **疊放只留第一顆(丟棄其餘、不用內側鍵)**:更省,但失去強調手感;疊放與單顆變得無法區分。

## 連帶約束

- 分組採**錨點制**(一顆只跟該疊放組的第一顆比 < 1/8 beat)以避免鏈式串接,確保可決定性。
- 收斂後的音符 `finger` 記為 `index`(內側鍵屬食指內側列),`bank` 取錨點的列。渲染(issue 06)需為內側鍵預留獨立道位。
